PHASE 1 — CRASH TRIAGE FIRST

The app is crashing in Replit. Find the exact crash source first.

Tasks:
1. Run dependency install:
   npm install
   or use the package manager already configured in the project.
2. Run:
   npm run build
3. Run dev server.
4. Capture all terminal errors.
5. Capture browser console errors if available.
6. Identify whether the crash is caused by:
   - missing dependency
   - wrong import path
   - broken TypeScript
   - env variable issue
   - Firebase config issue
   - Capacitor-only code running in browser
   - Android plugin import breaking web build
   - bad route
   - broken component render
   - undefined state/store data
   - duplicate file/conflict from migration
   - malformed JSON/config file
7. Fix only the crash root cause.
8. Add a React Error Boundary if the app does not already have one, so a single component crash does not blank the entire app.

React error boundaries are appropriate here because they catch render-time component errors and show a fallback UI instead of crashing the whole tree.
